# Контрольная работа: Простое ToDo приложение

## Цель работы
Создать максимально простое ToDo приложение, используя все навыки, полученные за предыдущие уроки (модели, миграции, function-based views, ModelForm, CRUD, поиск и фильтрация).

## Модель данных

Необходимо создать модель `Task` со следующими полями:

- `title` — CharField (название задачи, обязательное поле)
- `description` — TextField (описание задачи, необязательное)
- `completed` — BooleanField (статус выполнения, по умолчанию `False`)
- `completed_at` — DateTimeField (Дата выполнения, необязательное)
- `created_at` — DateTimeField (дата создания, `auto_now_add=True`)

## Обязательная функциональность

### 1. Список задач (главная страница)
- Отображение всех задач
- Показ названия задачи, статуса (выполнена / не выполнена) и даты создания
- Показ completed_at если выполнена
- **Поиск** по названию задачи
- **Фильтр** по статусу:
  - Все задачи
  - Только выполненные
  - Только невыполненные

### 2. Создание новой задачи
- Форма для создания задачи
- После успешного создания — редирект на главную страницу
- Валидация формы

### 3. Редактирование задачи
- Возможность изменить название, описание и статус выполнения существующей задачи
- Использовать `ModelForm` с передачей `instance`
- После сохранения — редирект на страницу задачи или главную

### 4. Удаление задачи
- Удаление задачи с подтверждением (отдельная страница подтверждения удаления)
- После удаления — редирект на главную страницу

### 5. Изменение статуса выполнения
- Возможность отметить задачу как выполненную или вернуть в невыполненные
- Если задача отмечается как выполненная добавлять completed_at
- Можно реализовать как: 
  - Отдельную кнопку/ссылку на странице списка, или
  - Через форму редактирования

## Требования к реализации

- Новый репозиторий GitHub
- Обьязательно скрывайте лишние файлы в .gitignore
- Без нейронок!
