# Итоговая контрольная работа по Django


## Задание

Реализовать небольшой проект — **«Заметки» (Notes)** — веб-приложение для хранения заметок пользователя.

Нужно сделать следующее:

1. Создать модель `Note` с полями:
   - `title` — заголовок заметки (строка, максимум 200 символов)
   - `text` — текст заметки
   - `created_at` — дата создания, заполняется автоматически
   - `is_important` — булево поле, важная заметка или нет (по умолчанию `False`)

2. Сделать и применить миграции. Зарегистрировать модель `Note` в админ-панели.

3. Сделать базовый шаблон `base.html` с блоком `{% block content %}{% endblock %}` и подключить его через `{% extends %}` во всех остальных шаблонах.

4. Реализовать через **Class-Based Views** полный CRUD для заметок:
   - `NoteListView` (`ListView`) — список всех заметок (заголовок + пометка, если важная), с ссылкой на каждую заметку
   - `NoteDetailView` (`DetailView`) — страница одной заметки со всеми полями
   - `NoteCreateView` (`CreateView`) — создание заметки через `ModelForm`, после сохранения редирект на список
   - `NoteUpdateView` (`UpdateView`) — редактирование заметки, после сохранения редирект на страницу заметки
   - `NoteDeleteView` (`DeleteView`) — удаление заметки с подтверждением, после удаления редирект на список
   - прописать `urls.py` для всех пяти view через `.as_view()`

5. Дополнительно сделать через **Function-Based View** страницу поиска заметок: обычная `forms.Form` (не ModelForm) с одним полем `query` (CharField). При GET — показываем форму. При POST — ищем заметки, у которых `title` содержит введённый текст (`Note.objects.filter(title__icontains=query)`), и выводим найденные в том же шаблоне.
